Recommended setup — app public, vault private
This app can be hosted publicly (e.g. GitHub Pages) while your financial data stays in a completely separate private repository. Only you can access the vault.
  1. Host the app publicly — fork or upload index.html to a public GitHub repo, enable GitHub Pages under Settings → Pages → Deploy from branch.
  2. Create a private vault repo — go to github.com/new, name it something generic (e.g. v-store), set it to Private, and initialise with a README so it exists.
  3. Create a fine-grained Personal Access Token — go to GitHub → Settings → Developer settings → Personal access tokens → Fine-grained tokens → Generate new. Set:
     · Resource owner: your account
     · Repository access: Only select repositories → choose your private vault repo only
     · Permissions: Contents → Read and write
    This token can read/write your vault repo and nothing else.
  4. Choose a strong passphrase — use 5 or more random words (diceware) or a random 20+ character string. This is the only thing protecting your data if someone downloads the encrypted file. Do not reuse a password from anywhere else.
  5. Fill in the form below — enter the token, passphrase, and the private vault repo owner/name. Click Test, then Open vault.
⚠ Your passphrase cannot be recovered. If you forget it, the vault file cannot be decrypted. Store it in a password manager.

Switch vault repository
Move your vault to a different GitHub repository — for example from a public repo to a private one. Your data is re-encrypted with your current passphrase and pushed to the new location. The old repository file is not deleted.
Your token must have Contents: read & write permission on the new repository. The vault file will be created as budget.enc. After switching, point your token scope to the new repo only.
